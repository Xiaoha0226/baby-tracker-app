## Context

宝宝记录应用使用 Vue 3 + TypeScript 构建前端，采用 Pinia 状态管理。顶部导航栏位于 Home.vue 页面，包含五个功能按钮：统计（📊 统计）、宝宝切换器、日期显示、设置（⚙️ 设置）、退出。当前布局使用 flexbox 固定间距，在小屏幕设备上可能存在按钮拥挤和日期文本溢出问题。

## Goals / Non-Goals

**Goals:**
- 实现顶部导航栏响应式布局，支持4.7至6.7英寸手机屏幕
- 日期显示区域支持文本自适应，确保完整显示
- 建立CSS断点系统，提供流畅的布局过渡
- 保持所有按钮的可点击区域不小于44x44px
- 在各屏幕尺寸下保持一致的用户体验和视觉美感

**Non-Goals:**
- 不修改现有路由结构
- 不改变功能按钮的业务逻辑
- 不实现暗色模式的响应式适配（已有独立实现）
- 不修改其他页面的布局

## Decisions

### 1. 断点策略
- **决策**: 采用移动优先的断点设计，定义三个关键断点
- **断点划分**:
  - `sm`: 320px-374px（4.7-4.9英寸）
  - `md`: 375px-413px（5.0-5.5英寸）
  - `lg`: 414px-767px（5.6-6.7英寸）
- **理由**: 覆盖主流Android和iPhone设备屏幕宽度

### 2. 日期显示处理
- **决策**: 日期容器使用 `min-width` 约束，允许收缩但不截断文本
- **策略**:
  - 默认使用 `white-space: nowrap` 保持单行
  - 容器设置 `overflow: visible` 确保可见
  - 按钮区域使用 `flex-shrink: 0` 保护最小尺寸
  - 日期区域使用 `flex: 1` 占据剩余空间但设置 `min-width`
- **理由**: 日期文本是用户感知当前日期的核心信息，不能被截断

### 3. 按钮布局策略
- **决策**: 在小屏幕使用两行布局，统计+宝宝为上行，日期为下行中央
- **布局变换**:
  - `lg+`: 所有按钮单行水平排列（当前实现）
  - `sm/md`: 上下两行布局，日期居中独占一行
- **理由**: 确保每个按钮有足够触控区域，避免误触

### 4. 触控区域保护
- **决策**: 按钮设置 `min-width: 44px` 和 `min-height: 44px`
- **理由**: 符合 Apple HIG 和 Google Material Design 触控区域指南

### 5. CSS实现方式
- **决策**: 使用 CSS Grid + Flexbox 混合布局
- **理由**: Grid 处理整体框架，Flexbox 处理按钮细节

## Risks / Trade-offs

- **[风险] 按钮文本在小屏幕上可能仍然过长** → [缓解] 使用 `text-overflow: ellipsis` 配合 `aria-label` 提供完整信息
- **[权衡] 两行布局增加页面高度** → 接受，因为确保了可用性更重要
- **[风险] Android WebView 兼容性问题** → [缓解] 使用标准 flexbox 属性，避免实验性特性

## Migration Plan

无需数据库迁移，纯前端样式变更。

部署步骤：
1. 更新 Home.vue 的 `<style scoped>` 样式
2. 更新 theme.css 添加响应式工具类
3. 在各设备上进行视觉验证
